# pispot

A Raspberry Pi 5 Wi-Fi hotspot and router with a live web dashboard.
The Pi shares an upstream Wi-Fi connection (USB adapter on `wlan1`) to
clients connected to its built-in hotspot (`wlan0`), with a browser-based
dashboard for monitoring and control.

## What it does

- **Live dashboard** — interface throughput, hotspot clients, WAN link
  status, admin/backup interface, system load/memory/temperature
- **WAN control** — WAN Up / WAN Down buttons start and stop the upstream
  wpa\_supplicant connection
- **WiFi network management** — add, remove, and apply WAN networks through
  the browser without touching the Pi
- **Role-based access** — read-only and admin roles via Unix groups and PAM;
  credentials sent over HTTPS with Basic Auth
- **HTTPS only** — TLS terminated in the dashboard container; cert and key
  mounted from the host

## Hardware

| Component | Role |
|---|---|
| Raspberry Pi 5 | Host |
| Built-in `wlan0` | Hotspot / LAN (AP mode, `10.10.10.1/24`) |
| USB Wi-Fi adapter `wlan1` | WAN uplink (station mode) |
| Ethernet `eth0` | Admin / backup WAN |

> Only the Raspberry Pi 5 has been tested. Other models are not supported.

## Architecture

```
clients ──── wlan0 (hostapd + dnsmasq)
                │
              Pi 5
                │
             wlan1 ──── upstream AP
             eth0  ──── admin network

┌────────────────────────────────────────┐
│  Docker container: pispot-ui           │
│  Go HTTP server, :443 HTTPS            │
│  reads /proc, /sys, dnsmasq leases     │
│  talks to pispot-authd via Unix socket │
└────────────────────────────────────────┘
                │ /run/pispot/pispot-authd.sock
┌────────────────────────────────────────┐
│  systemd service: pispot-authd (root)  │
│  PAM authentication via pamtester      │
│  WAN control via systemctl + dhcpcd    │
│  WiFi config via wpa_supplicant conf   │
└────────────────────────────────────────┘
```

**pispot-ui** runs in Docker with host networking and a read-only rootfs.
It never runs as root and has no write access to the host.

**pispot-authd** runs as a native systemd service with root privileges.
It is the only component that touches wpa\_supplicant, dhcpcd, and PAM.
The dashboard delegates all privileged operations to it over a Unix socket.

## Prerequisites

**Mac (build machine):**
- Go 1.26+
- Docker Desktop (arm64 image builds require no emulation on Apple Silicon)
- Ansible (`pip install ansible` or `brew install ansible`)
- Python 3.9+
- Node.js (for `make js-check`)

**Raspberry Pi 5:**
- **Raspberry Pi OS Lite (64-bit)** — no desktop, minimal footprint.
  Flash with [Raspberry Pi Imager](https://www.raspberrypi.com/software/).
- Docker 29+
- `pamtester` (installed by `make setup`)

## First-time setup

### 1. Prepare the Pi

1. Flash **Raspberry Pi OS Lite (64-bit)** to a new SD card using Raspberry Pi Imager
2. In Imager's **Edit Settings** before writing:
   - Set the hostname
   - Enable SSH
   - Create your user account and password
3. Boot the Pi. If you set up password-only auth, copy your SSH key:
   ```sh
   ssh-copy-id <user>@<hostname>.local
   ```
4. Confirm SSH access before proceeding:
   ```sh
   ssh <user>@<hostname>.local
   ```
5. Note the hostname and username — you will enter both in `make init`

### 2. Clone and configure

```sh
git clone https://github.com/mcs-net/pispot
cd pispot
make init
```

The wizard prompts for Pi hostname, SSH user, dashboard title, TLS cert
name, HTTPS hostname, Wi-Fi country, hotspot credentials, and WAN network
credentials. It writes three gitignored files:

- `dashboard/.env` — Pi hostname, cert name, dashboard title
- `ansible/inventory/hosts` — Ansible inventory
- `ansible/group_vars/all.yml` — all site-specific variables

Re-running `make init` pre-populates every prompt with the saved value;
press Enter to keep it unchanged.

### 3. TLS certificate

Place your certificate files in `ssl/` using the cert name you chose:

```
ssl/<cert-name>.crt    certificate
ssl/<cert-name>.key    private key
ssl/<cert-name>.ca     CA/intermediate bundle
```

For local testing, generate a self-signed cert instead:

```sh
make gen-certs
# then set PISPOT_TLS_CERT_NAME=test_cert in dashboard/.env
```

### 4. Build

```sh
make build          # compile pispot-authd, pispotctl (linux/arm64)
make docker-build   # build pispot-ui Docker image (linux/arm64)
```

### 5. Bootstrap the Pi (first time only)

```sh
make bootstrap      # apt upgrade, install packages, Docker, set Wi-Fi country
```

> When prompted `BECOME password:`, enter the sudo password for your Pi user account.

This installs Docker from the official Docker apt repository and adds your
user to the `docker` group. The Pi reboots if required — the reboot also
ensures the new group membership is active for subsequent steps.

### 6. Deploy

```sh
make deploy-all
```

> When prompted `BECOME password:`, enter the sudo password for your Pi user account.

This will:
1. Build host binaries (`pispot-authd`, `pispotctl`)
2. Run Ansible `pi-setup.yml` — creates pispot Unix groups, adds your user
   to `pispot-admin`, configures dhcpcd, hostapd, dnsmasq, nftables, avahi,
   installs pispot-authd and pispotctl, deploys TLS certs
3. Build the Docker image and ship it to the Pi
4. Start the dashboard container

A confirmation prompt is shown before proceeding, as the deploy restarts
all services and briefly disconnects active dashboard users.

## Day-to-day operations

```sh
make deploy-dashboard   # rebuild and redeploy the dashboard only
make deploy-authd       # rebuild and redeploy pispot-authd + pispotctl only
make checkup            # read-only Ansible health check
make logs               # ssh to Pi and run pispotctl logs
make ssh                # open an SSH session to the Pi
```

> `deploy-authd` and `checkup` run Ansible and will prompt `BECOME password:` —
> enter the sudo password for your Pi user account.

## User accounts and access

Two Unix groups control access:

| Group | Role | Permissions |
|---|---|---|
| `pispot-ro` | read-only | view dashboard, no controls |
| `pispot-admin` | admin | full dashboard, WAN up/down, WiFi config |

Both groups are created automatically by `make setup`. The user you entered
in `make init` is automatically added to `pispot-admin` on first deploy.

To add additional users on the Pi:

```sh
sudo useradd -m alice
sudo passwd alice
sudo usermod -aG pispot-ro alice      # read-only
sudo usermod -aG pispot-admin alice   # or admin
```

Credentials are validated via PAM on each login (cached for 5 minutes).

## pispotctl

The `pispotctl` tool is installed at `/usr/local/bin/pispotctl` on the Pi
and requires root for all commands.

```sh
sudo pispotctl wan-up        # start wpa_supplicant@wlan1 + request DHCP
sudo pispotctl wan-down      # stop wpa_supplicant@wlan1 + release DHCP
sudo pispotctl status        # show service states and network info
sudo pispotctl wifi list     # list configured WAN networks (passwords masked)
sudo pispotctl logs          # dump all pispot-related logs
```

## Dashboard environment variables

These are set in `dashboard/docker-compose.yml` and sourced from
`dashboard/.env`. The `.env` file is generated by `make init`.

| Variable | Default | Description |
|---|---|---|
| `LISTEN_ADDR` | `:443` | Listen address |
| `TLS_CERT_FILE` | | Path to fullchain.pem inside container |
| `TLS_KEY_FILE` | | Path to private key inside container |
| `REQUIRE_TLS` | `true` | Reject non-TLS starts |
| `AUTH_SOCKET` | | Unix socket path to pispot-authd |
| `AUTH_REALM` | `pispot` | HTTP Basic Auth realm |
| `DASHBOARD_TITLE` | `pispot` | Page title and header text |
| `HOTSPOT_IF` | `wlan0` | LAN / AP interface |
| `WAN_IF` | `wlan1` | WAN / upstream interface |
| `ADMIN_IF` | `eth0` | Admin / backup WAN interface |
| `PROC_PATH` | `/host/proc` | Host /proc mount point |
| `SYS_PATH` | `/host/sys` | Host /sys mount point |
| `LEASES_PATH` | `/host/dnsmasq.leases` | dnsmasq lease file |

## Make targets reference

```
Build:
  make build              build-authd + build-pispotctl (linux/arm64 binaries)
  make build-dashboard    compile pispot-ui locally (Mac, syntax check only)
  make build-authd        cross-compile pispot-authd → host/files/pispot-authd
  make build-pispotctl    cross-compile pispotctl → host/files/pispotctl

Test:
  make test               go test ./...
  make js-check           node --check on all embedded JS files
  make vet                go vet ./...
  make fmt                gofmt -w .

Docker:
  make docker-build       build pispot-ui:latest image (linux/arm64)
  make image-size         print local image size

Deploy:
  make ship-dashboard     save + gzip + ssh + docker load image onto Pi
  make restart-dashboard  pipe compose over SSH, docker compose up -d --no-build
  make deploy-dashboard   docker-build + ship-dashboard + restart-dashboard
  make deploy-authd       build-authd + build-pispotctl + Ansible pi-setup
  make deploy-all         full stack deploy with confirmation prompt

Ansible:
  make bootstrap          one-time base setup (apt, packages, Wi-Fi country)
  make setup              full OS configuration
  make checkup            read-only health check

Init / first-time:
  make init               interactive wizard → dashboard/.env, hosts, all.yml
  make gen-certs          generate self-signed test TLS certs
  make check-secrets      verify required files exist before deploy

Pi management:
  make logs               ssh to Pi and run pispotctl logs
  make ssh                open SSH session to Pi
```
